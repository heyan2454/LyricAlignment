## Superseding vowel-boundary decision

On 2026-07-22 the user accepted the vowel-boundary v2 grouping as the new M4Singer working contract, replacing the earlier 6,051 accepted / 14,845 review contract.  The v2 rule keeps consecutive identical vowel/final tokens within a held-vowel candidate, but starts a new candidate at either a new initial or a different vowel/final; a different vowel without an initial is retained as a zero-initial candidate rather than being absorbed into the preceding character.

The corresponding full rerun is external at `/home/hyan/Data/lyricalign/derived/20260722_m4singer_vowel_boundary_v2`: 20,896 total = 19,501 accepted + 1,395 review.  It records 13,450 review-to-accepted transitions versus the former grouping.  The review distribution is: multiple/complex slur with missing syllable groups 892; single slur with one missing syllable group 294; zero-initial or special-syllable case 173; multiple note groups per character 27; missing syllable groups 8; latin/digit 1.

Unresolved by explicit user direction: repeated finals in the `i**` and `u**` families, plus repeated `an`, `a`, `ou` and related finals, are intrinsically ambiguous under current metadata.  A repeated token can encode a held/slurred vowel or a separate lyric character.  No fixed allocation ratio, heuristic reassignment, or automatic acceptance change has been applied for this ambiguity.  A later review must use auditable evidence (for example manual samples, additional annotation provenance, or a separately validated rule) before refining it.
